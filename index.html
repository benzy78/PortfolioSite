UI/UX Designer based in Japan, creating seamless digital experiences.

Introduction

Yuki Nishimura ｜UI/UX Designer

慶應義塾大学経済学部卒。学生時代からインターンやフリーランスとして、LPやWebサイト、Webアプリの制作に携わり、主にコーディングとUI/UXデザインを担当してきました。プロジェクトごとに異なる課題やユーザー層に向き合いながら、柔軟かつ論理的に設計・改善を行ってきた経験があります。

UI/UX設計においては、単に視覚的に美しいものを作るだけでなく、ユーザーの課題を深く理解し、それをどう解決するかという観点から情報設計とUI設計を行うことを重視しています。操作しやすく、迷わない導線設計や、利用文脈に即したコンポーネント設計を心がけ、ユーザー体験の質を高めることを常に意識しています。

また、アプリケーションのように継続的に成長していくプロダクトでは、再利用性・保守性・拡張性といった観点からのデザインシステムの構築が重要だと考えており、長期的にプロダクトと向き合えるUIの土台作りにも取り組んでいます。 こうした設計力をさらに深めるためにも、必要な知識や技術は自ら学び、継続的にアップデートしていく姿勢を大切にしています。

Works

1.Smart Ordee

SmartOrdee（スマートオーディー）は、飲食店の注文体験をスムーズにするQRコードオーダーシステムで、そのUIデザインを制作しました。本プロジェクトはポートフォリオ用に制作した架空のサービスを想定したものであり、実際の提供サービスではありません。ただし、飲食店での実体験をベースにUX課題を洗い出し、実運用を想定したユーザー導線・機能設計、デバイスごとのUI最適化、視認性や操作性の検証などを通じて、実際のリリースにも耐えうる品質を目指して制作しました。

  • 制作期間：２ヶ月弱
  • 使用ツール：Figma
  • 担当範囲：UX設計、UI設計、デザインシステム構築
  • ※本プロジェクトはポートフォリオ用に個人で制作したもので、全工程を一人で担当しています。
Figmaで見る

＊実際のUIプロトタイプがご覧いただけます。

1. 背景・動機

日常的に飲食店で利用するQRコードオーダーシステムに対して、「UIが直感的でない」「情報が見づらい」「操作のフィードバックが曖昧」「誤操作しやすい」といった課題を感じたことが、本アプリをデザインするきっかけとなりました。そこで、自分なりに改善の余地があるポイントを洗い出し、よりスムーズでストレスのない注文体験を実現できるサービスとしてSmartOrdeeを設計しました。

2. リサーチ・ターゲット

まず、競合サービスの分析をすると、情報設計やUIUXの精度に課題が多く見られました。たとえば、カテゴリー構造が複雑で情報が階層化されておらず、メニューの違いが見分けづらい、検索アイコンの意味が分かりにくい、画面幅の制約で一覧性が損なわれるなど、ユーザーにとっての「迷い」や「混乱」が多く発生している状況が明らかになりました。SmartOrdeeでは、こうした課題に対して情報の階層整理・一覧性の向上・直感的な操作導線の設計などを通じて、幅広いユーザーが使いやすい体験を目指しました。

ターゲットは、中〜小規模の飲食店を想定しており、低コストで導入でき、業務効率化や売上分析にも活用できる注文システムを求める店舗に向けて設計しました。エンドユーザーに関しては、デジタル機器に不慣れな中高年層でも迷わず使えるUIUXを意識し、「分かりやすさ」と「安心感」のある操作体験を重視しています。

3. ユーザーフロー

ユーザー画面は、飲食店に来店するお客様が初めて操作するケースも多いことを前提に、インタラクションコストの削減と迷いのない導線設計を重視しました。視認性を高め、不要な情報を排除する一方で、各操作には視覚的なフィードバックを付与することで、「自分の操作が正しく反映されたか」が直感的に分かるよう配慮しています。

また、UX設計上よく問題となる「分かりやすさ」と「使いやすさ」のトレードオフが発生する場面においては、初回利用者が多いという特性を踏まえ、分かりやすさを優先しています。システムに慣れていないユーザーでも戸惑わずに注文できるよう、意図的に操作手順を簡略化しすぎず、丁寧にガイドする構成としました。

一方、管理画面は業務で繰り返し使うアクティブユーザー（店舗スタッフ）向けであるため、分かりやすさよりも操作効率や情報の一覧性を重視しています。頻繁に行う操作には最小限のステップでアクセスできるよう設計し、複数業務を並行して行いやすい構成としました。

4. UI設計の意図

本アプリのUIは、「誰でも直感的に使えること」を前提に、要素の構成・視線誘導・タイポグラフィー・配色・余白といったビジュアル設計の基本要素を丁寧に設計しました。

ユーザー画面では、必要な情報だけに絞り込みながらも、視認性・階層構造の明確さを意識して、情報密度が高くなりすぎないように余白をしっかり確保。また、明瞭な文字サイズとフォントの選定により、視力や年齢によらず読みやすい画面を実現しています。配色設計においては、アクセントカラーを限定的に使い、重要なアクションや選択状態が視覚的に際立つように調整しました。ボタンやインタラクティブな要素は、十分なコントラストとサイズ（44px × 44px以上）を確保し、快適な操作感と誤操作防止の両立を図っています。

管理画面では、一画面での情報把握や、操作のスピード感を重視したUI設計とし、カードやテーブルなどの構成要素を適切に整理。パネルやボタンの視覚的な区切りや、階層ごとのコントラスト設計により、視線の動線が自然に流れるように設計しています。

全体としては、独創的なビジュアルや装飾的な演出よりも、使い慣れていない人でも違和感なく利用できる「整然としたデザイン」を目指し、UIとしての完成度とプロダクトとしての一貫性を両立しています。

5. 工夫した点・改善点

工夫した点：UI/UX設計に加えて、Figma上でのデザインシステム構築にも注力しました。スタイルガイドの作成、適切なコンポーネント化、カラー・タイポグラフィ・エフェクトのバリアブル管理を通して、再利用性・拡張性・保守性・一貫性の高いデザイン運用を意識。ユーザーには直接見えない部分ながら、長期的なプロジェクト運用を見据えた“土台づくり”に取り組みました。

改善点：本アプリはポートフォリオ用に制作したものであり、実際の運用環境でのユーザーテストやフィードバックは行えていません。そのため、現時点ではUIUX上の明確な課題は見えていないものの、実運用では利用者の行動データや声を通じて初めて見えてくる課題も多いと考えています。今後、実際のプロダクト開発に関わる中では、そうしたリアルな利用状況に基づいた検証と改善のサイクルを意識し、事前に想定しきれない問題にも柔軟に対応できる設計力と姿勢を大切にしていきたいと考えています。

2.Smart Ordee紹介用LP

本LPは、飲食店向けQRコードオーダーサービス「SmartOrdee」の魅力を伝えるための紹介用ランディングページとして設計・制作したものです。サービスの機能やUI設計のポイントを、視覚的にも分かりやすく伝える構成とし、ユーザーが利用イメージを具体的に想像できるような導線設計を意識しています。

  • 制作期間：1週間
  • 使用ツール：Figma, HTML, Sass, JavaScript, Git, Github
  • 担当範囲：UX設計、UI設計、コーディング
  • ※本プロジェクトはポートフォリオ用に個人で制作したもので、全工程を一人で担当しています。
SmartOrdeeのLPを見る

1. 背景・動機

アプリ本体のデザインを通して、「誰でも迷わず使える注文体験」を追求してきましたが、それをどのように伝えるかも同様に重要だと考え、紹介LPの制作に取り組みました。実際のサービスサイトに近い構成で、見た目だけでなく、体験価値や使いやすさの伝達を目的とした設計にしています。

2. リサーチ・ターゲット

個人経営の飲食店（初期費用を抑えたい・人手不足の解消・効率化を目指す店舗）、中小規模のレストランチェーン・カフェ・居酒屋など（回転率の向上や注文ミスを減らしたい店舗）をペルソナとして想定し、機能・導入メリットが直感的に理解できる構成としています。

＊リサーチに関しては、上述のアプリの部分と重複するので省略しました。

3. ユーザーフロー

LP全体の構成は、「サービスを理解し、納得し、行動につながる」流れを意識して設計しました。初めにサービスの概要を簡潔に提示し、次に導入メリット（注文率アップ・業務効率化など）をわかりやすく訴求。続く導入事例では、実際の効果を数値とともに紹介することで、信頼性と説得力を補強しています。

その後、よくある質問セクションで導入前の不安や疑問を解消し、最後に無料トライアルまたは資料請求の問い合わせへとスムーズに誘導。一貫した視線誘導と心理的導線設計により、ユーザーが自然な流れで行動を起こせる構成としています。

4. UI設計の意図

本LPでは、「SmartOrdee」の魅力を明確かつ効果的に伝えること、そしてアプリ登録者を増やすことを目的に、UI設計を行いました。配色には飲食店に親和性の高い落ち着いた赤系カラーを採用し、信頼感と食欲を自然に喚起しながら、モダンで洗練された印象を与えるよう、構成要素やUIコンポーネントを設計しています。

また、視線の流れに配慮したタイポグラフィと適切な余白設計により、アプリの訴求点を端的に伝えることを重視。コンバージョン率向上を意識し、「ちょっと使ってみたい」と思わせるタイミングでCVボタンを配置するなど、行動喚起の導線設計にも工夫を施しました。

加えて、離脱率の低下のため、ビジュアルを要所に配置しつつ、テキスト情報とのバランスに配慮し、読んでいて疲れないような構成にしています。

5. 工夫した点・改善点

工夫した点：「見た目の美しさ」よりもアプリ登録という目的の達成を最優先に設計しました。導入メリットを的確に伝えるためのタイポグラフィ・配色・情報設計、そして行動を促す導線設計（CVボタンの配置など）を一貫してUX視点で構築しています。特に、ユーザーが「少し使ってみたいかも」と感じるタイミングを意識したCV導線の設計は、効果的な訴求を意識したUI設計の一例です。

改善点：導入メリットの訴求については、テキスト中心ではなく図解やビジュアルを用いた構成にすることで、より直感的な理解や印象付けができたかもしれないと感じており、今後の改善ポイントとして意識しています。

3.Eco Hybe HP

EcoHybe（エコハイブ）」は、リユース・リメイク・サステナブルをテーマにした架空のリサイクルショップのコーポレートサイトです。地域に根ざしたリサイクルショップを想定し、親しみやすさと温かみが伝わるよう、柔らかい配色やタイポグラフィを中心にデザインを構成しました。実在の企業ではなく、ポートフォリオの表現幅を広げることを目的に制作した架空サイトです。

  • 制作期間：1週間
  • 使用ツール：Figma, HTML, Sass, JavaScript, Git, Github
  • 担当範囲：UX設計、UI設計、コーディング
  • ※本プロジェクトはポートフォリオ用に個人で制作したもので、全工程を一人で担当しています。
Eco Hybe HPを見る

1. リサーチ・ターゲット

本プロジェクトでは、セカンドストリートやブックオフなどの既存リユースショップのWebサイトを調査し、構成やUI、ブランドの打ち出し方を比較しました。多くの競合サイトは、価格や買取強化などビジネス的な訴求に偏っており、環境意識やライフスタイルへの共感を軸にした設計は弱いと感じました。そこで、EcoHybeでは、「リユースは未来をつくる選択肢」というブランドメッセージを掲げ、単なる取引の場ではなく、共感や価値観の共有を大切にした体験設計を目指しました。

ターゲットは、家具・家電・ファッションをお得に購入したい生活者、引越しなどを機に不要品を気軽に売りたい人、そしてサステナビリティに関心を持ち、リユースを日常に取り入れたい層としています。これらのユーザーが安心して利用できるよう、直感的でやさしい導線と、洗練されたUIでブランドイメージを構築しました。

＊本サイトはポートフォリオ用に制作した架空のサイトのため、背景や課題設定は省略し、構成や設計意図を中心にご紹介します。

2. ユーザーフロー

本サイトは、すぐに買取依頼や来店を促すことよりも、EcoHybeという会社がどんな理念・サービスを持つリサイクルショップなのかを伝えることに主眼を置いて設計しています。初めて訪れたユーザーが、「信頼できるお店だ」と感じられるよう、ブランドイメージや事業内容が端的に伝わる構成とし、サービス紹介→取扱カテゴリ→利用の流れ→企業理念→よくある質問→店舗情報といった流れで、自然に“知る→理解する→関心を持つ”へと導くユーザーフローを設計しました。また、視認性や情報の整理にも配慮し、「何ができるお店か」が迷わず伝わるようにナビゲーションや見出しの構成も工夫しています。

3. UI設計の意図

UI設計では、「ちょっと立ち寄ってみたい」と思えるような親しみやすさと温かみのある印象を重視しました。メインカラーにはリサイクルや自然を連想させる柔らかい緑系のカラーを選定し、フォントにはゴシックや明朝ではなく、角の取れた丸みのある書体の使用や、アイコンを多く利用することで、堅苦しさのない雰囲気を意識しています。また、サイトの第一印象となるメインビジュアルには実際の店舗外観写真を用い、各所に商品写真も配置することで、「ここなら気軽に行けそう」とイメージしてもらいやすいUIとなるよう設計しました。

4. 工夫した点・改善点

工夫した点：「親しみやすく、地域に根ざしたエコなリサイクルショップ」というブランドイメージを伝えるため、カラーとタイポグラフィーの選定に特にこだわりました。緑系の柔らかい配色と、角の丸いフォントを採用することで、やさしさや気軽さを視覚的に演出し、“エコに関心のある生活者が安心して利用できる”空気感を大切にしています。また、店舗外観や商品の写真を多く取り入れることで、実際の店舗の雰囲気をイメージしやすくし、記憶に残るビジュアル体験を意識しました。「なんとなく覚えていて、近くに寄ったら立ち寄ってみたくなる」──そんな感覚を引き出すUIを目指しています。

改善点：サイト全体の導線において、具体的な行動を促すフックがやや弱いと感じています。例えば、「買取までの流れ」や「利用者の声」など、初めての人が次のステップをイメージしやすくなる要素があると、より行動につながるサイトになると考えています。今後は、「親しみやすさ」に加えて、「はじめの一歩を踏み出しやすい情報設計」も強化していきたいです。

4.Boulangerie Amande HP

「Boulangerie Amande（ブーランジェリーアマンド）」は、高級志向の海外風ベーカリーを想定して制作した、架空の店舗サイトです。ブランドの世界観を丁寧に表現するため、洗練されたタイポグラフィや配色、余白設計にこだわり、特別感のあるビジュアル体験を意識しました。本サイトはポートフォリオ用に制作したもので、UI/UX設計力やブランド表現力を伝えることを目的としています。

  • 制作期間：1週間
  • 使用ツール：Figma, HTML, Sass, JavaScript, Git, Github
  • 担当範囲：UX設計、UI設計、コーディング
  • ※本プロジェクトはポートフォリオ用に個人で制作したもので、全工程を一人で担当しています。
Boulangeie AmandeのHPを見る

1. リサーチ・ターゲット

本プロジェクトでは、洗練された高級感を持ちながらも、日常に少しの特別感を添えるようなパン屋のブランドイメージを目指し、Webサイトを設計しました。類似ジャンルの店舗サイトやカフェ系ブランドのLPなどを参考に、上質感・信頼感・親しみやすさのバランスをリサーチし、過度な高級志向に寄りすぎず、あくまで“手に届く特別感”を意識したトーンを検討しました。

メインターゲットは、30〜50代の女性。パンやカフェが好きで、SNSを通じておしゃれなグルメ情報をチェックするような層です。食材の質や安全性にも敏感で、無添加やオーガニックといったキーワードにも価値を感じています。Boulangerie Amandeでは、そうしたライフスタイルに共鳴するデザインを意識し、視覚的な高級感と素材へのこだわりが伝わる構成としました。

＊本サイトはポートフォリオ用に制作した架空のサイトのため、背景や課題設定は省略し、構成や設計意図を中心にご紹介します。

2. ユーザーフロー

本サイトはオンライン購入などの即時的なコンバージョンを目的とした構成ではなく、「お店の世界観を伝え、行ってみたいと思ってもらうこと」を重視したブランド訴求型の構成としています。ユーザーがサイトに訪れてまず目にするのは、素材や製法へのこだわりを伝えるコンテンツ。そこから、魅力的な商品紹介へと導線をつなぎ、店舗の雰囲気や商品の特長を視覚的に感じてもらう設計としています。さらに、アクセス情報や営業時間などの実用的な店舗情報、新商品や営業日を伝えるお知らせセクションを配置し、ユーザーが自然な流れで「行ってみよう」と思えるような体験を設計しています。

3. UI設計の意図

高級志向のベーカリーというブランドコンセプトを反映し、UI設計では洗練された印象と信頼感を与えるビジュアル設計を意識しました。フォントには明朝体やセリフ系を使用し、クラシックで上質な空気感を演出。配色は黒を基調に、アクセントとしてワインレッドとイエローを採用し、高級感と温かみのある飲食店らしさの両立を図りました。また、店内やパン、職人の写真を要所に取り入れ、「空気感が伝わるようなビジュアル」と必要なテキスト情報とのバランスにも配慮。商品のクオリティだけでなく、店舗全体の雰囲気や世界観が伝わるUIを目指して設計しています。

4. 工夫した点・改善点

工夫した点：UI設計においては、ブランドの世界観を成立させるだけでなく、その印象を一貫して保ち続けるための「細部の設計」に注力しました。特に、写真とテキストの配置バランスや余白の取り方、アクセントカラーの使用箇所などを、視線の流れや情報の強弱に合わせて丁寧に調整し、単に「高級感がある」だけでなく、ストレスなく読み進められる体験を意識しました。また、装飾を最小限に抑えることで、主役である商品のビジュアルが自然と引き立つようなUI設計を心がけています。

改善点：UI全体がブランド訴求に寄っているぶん、「ユーザーが次にどう行動すればいいか」の情報設計がやや控えめだった点が挙げられます。たとえば、「よくある質問」や「予約に関する案内」など、来店前の不安を解消する要素をもう少し補強してもよかったかもしれません。今後は、ブランドイメージとユーザー行動のバランスをさらに最適化する視点も取り入れていきたいと考えています。

5.OrgaTask 紹介LP

本サイトは、架空のタスク管理アプリ「OrgaTask」の機能や魅力を紹介し、ユーザー登録へと自然に誘導することを目的としたランディングページです。個人制作プロジェクトとして、サービスの世界観やメリットを視覚的かつ端的に伝える構成・デザインを意識して制作しました。ファーストビューではアプリの価値を直感的に伝え、セクションごとに特徴・利用シーン・導入メリットを明示することで、興味を持ったユーザーが迷わずアクションを起こせる導線設計としています。

  • 制作期間：1週間
  • 使用ツール：Figma, HTML, Sass, JavaScript, Git, Github
  • 担当範囲：UX設計、UI設計、コーディング
  • ※本プロジェクトはポートフォリオ用に個人で制作したもので、全工程を一人で担当しています。
OrgaTaskのLPを見る

1. リサーチ・ターゲット

リサーチでは、既存のタスク管理ツールを調査し、多機能であるがゆえにUIが煩雑で、直感的に使いづらいと感じる場面が多いことを課題として捉えました。そこで、OrgaTaskでは「シンプルに使い始められ、使うほどに整理されていく」体験を訴求する構成を重視しています。

ターゲットは、業務やプロジェクトを並行して進行するオフィスワーカーや、家庭と仕事を両立しながらスケジュール管理が求められるワーキングマザー、複数クライアントの案件・納期を個別に把握したいフリーランスなど、幅広い職種・立場のタスク管理ニーズに対応できるユーザー像を想定しています。また、課題や授業、アルバイトなど、ライフスタイルが流動的な学生層も含め、使う人の生活に寄り添えるシンプルでフラットな導線設計を意識しました。

＊本サイトはポートフォリオ用に制作した架空のサイトのため、背景や課題設定は省略し、構成や設計意図を中心にご紹介します。

2. ユーザーフロー

初めて訪れたユーザーがアプリの魅力を瞬時に理解し、「試してみたい」と思えるような情報設計を意識しました。最初にアプリの世界観と機能性を視覚的に訴求するヒーロービジュアルを配置し、その後、「どのような課題を抱える人に向いているのか」「その課題をどう解決できるのか」といったストーリーラインで構成しています。ページをスクロールするごとに、特徴や導入メリット、利用シーンが整理された形で展開され、ユーザーが共感しながらアプリの価値を理解できる構造としました。また、ページ内には登録ボタンを複数箇所に配置し、興味を持ったタイミングですぐにアクションへ移れるよう導線を工夫しています。

3. UI設計の意図

UI設計では、訪問者がアプリの魅力を直感的に理解し、登録という行動へスムーズにつながる体験を意識しました。ファーストビューには、アプリのUIイメージとキャッチコピーを組み合わせて配置することで、サービスの内容がひと目で伝わる構成としています。

配色は、全体に信頼感と落ち着きを与えるブルー系をベースに使用し、背景や余白とのコントラストを調整することで、視認性と読みやすさを両立させました。その中で、CTAボタンのみはブルーの補色に近いオレンジ系のカラーを用いることで、視線を引きつけるアクセントとして機能させています。これにより、ユーザーの行動ポイントを視覚的に明示しつつ、ページ全体のトーンを崩さずに印象付けることができました。

また、情報のまとまりやセクションの切れ目が直感的に分かるように、ホワイトスペースを多めにとり、余白や要素間の距離にも配慮しました。ボタンや見出しといったインターフェース要素は、サイズ・色・位置の統一感を保ち、視線の流れを妨げないよう設計しています。ユーザーにとって負担の少ない、すっきりとしたUI体験を意識した構成です。

4. 工夫した点・改善点

工夫した点：ブランドイメージの演出よりも「CV率の向上」を最重要目標として据え、その目的に対して最適な構成を意識しました。特に、アプリの利便性や強みがユーザーにとってどのように役立つのかを具体的に伝えることで、「使ってみたい」と思わせる流れを意識して設計しています。ファーストビューから登録ボタンまで、どの情報をどの順序で提示すればユーザーの興味を維持できるかを丁寧に検討し、導線や視線誘導にも工夫を加えました。結果として、ビジュアルだけでなく、コンテンツの設計そのものがコンバージョンを意識した設計となっています。

改善点：アプリの特徴紹介において、使用シーンの具体性がやや弱く、ユーザーにとっての活用イメージが伝わりづらいと感じました。たとえば、「進捗を可視化してプロジェクト管理がスムーズになる」「課題や予定を整理して学業との両立を支援する」など、具体的なシーンや数値、イラストを交えた表現にすることで、訴求力を高められたと考えています。 また、「利用者の声」セクションには、顔写真やイラストといった視覚的な補強が不足しており、使用者像が曖昧な印象を与えてしまった点も課題です。視覚要素を加えることで、親近感と信頼感が増し、結果としてCV率の向上にもつながると感じました。

6.Hikora Design HP

本サイトは、Web制作会社「Hikora Design」を想定したポートフォリオ用の架空コーポレートサイトです。「見た目の美しさだけでなく、成果の美しさも。」をコンセプトに、Webサイト制作・SNS運用・保守管理などのサービスを提供する企業を想定し、構成・デザイン・UIを制作しました。実際の事業者に近いリアリティを持たせることで、実務レベルでの情報設計・デザイン設計力を表現することを目的としています。

  • 担当範囲：UX設計、UI設計、コーディング

1. リサーチ・ターゲット

本サイトは、会社の信頼感を伝える顔としての役割と同時に、新規顧客の集客につなげることを目的に設計しています。ターゲットは、ホームページを持ちたいと考えているものの費用面でハードルを感じている個人事業主や小規模団体。特に、名刺代わりのシンプルなWebサイトが必要な事業者や、事業初期段階で外注の選定に迷っている層を想定しています。

Web制作業界は現在、過剰なアニメーションや装飾による“自己満足型”のデザイン会社や、高額すぎる老舗企業、そして品質や対応力に不安の残るクラウドソーシングの出品者層に二極化している状況があります。そこで、Hikora Designでは、実務に基づいた確かな技術力と、生成AIを含む最新ツールの活用による工数削減を両立させ、リーズナブルかつクオリティの高い制作を求めるスモールビジネス層に向けたサービス提供を想定しています。

＊本サイトはポートフォリオ用に制作した架空のサイトのため、背景や課題設定は省略し、構成や設計意図を中心にご紹介します。

2. ユーザーフロー

本サイトは、初見ユーザーが最短距離で「信頼できそう」「相談してみよう」と思えるよう、シンプルかつ端的に会社の魅力を伝える導線を意識して構成しています。サイト訪問者はまず、ファーストビューで「実績と想い」を簡潔に掴み、その後、強み（低価格の理由・最新技術による工数削減・技術力の裏付け）を一目で理解できる構成に。その後、サービスの内容や制作事例を通して、「自分にも頼めそう」「ちゃんと成果を出してくれそう」と感じられるように設計。疑問や不安を抱かせず、“不明点がある前に問い合わせたくなる”導線を意識し、CTA（問い合わせ）ボタンも視線の流れの中で自然に配置しています。

3. UI設計の意図

本サイトでは、コーポレートサイトなので「信頼感」と「高級感」のある印象を与えることを重視し、UI設計を行いました。メインカラーには深いネイビーを採用し、全体のトーンも落ち着きのある色味で統一。フォントは明朝体およびSerif体を使用することで、理知的で品のある印象を意識しています。また、過剰なアニメーションや視覚効果はあえて排除し、余白やシャドウ、アイコンの使い方を通じて、静かに洗練された佇まいを目指しました。情報の伝わりやすさを損なわず、ビジネス目線でも「信頼できる会社」と感じられるUIを意識しています。

4. 工夫した点・改善点

工夫した点：「信頼できる制作会社であること」を静かに伝えるUIを目指し、装飾や演出に頼らず、色と文字だけで印象をコントロールする設計に挑戦しました。配色は深みのあるネイビーを基調に、余白やシャドウの使い方にも配慮し、高級感と落ち着きを両立した視覚表現を意識。また、フォントに明朝体とSerif体を組み合わせることで、信頼・知性・安定感といった企業イメージをタイポグラフィで補完する構成としました。

改善点：シンプルさを重視した結果、やや情報の“温度”が不足し、無機質に見える可能性もあると感じています。社員の写真や代表メッセージなど、“人”の要素を取り入れることで、問い合わせのハードルを下げ、より親しみやすさと信頼感を補完できたかもしれません。今後は、「静かな高級感」と「人のぬくもり」のバランスを取る設計も意識していきたいと考えています。
