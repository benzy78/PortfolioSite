UI Design

  • Figma

    オートレイアウト、変数、コンポーネント/Variants を活用し、拡張性・一貫性を備えたデザインシステムと UI ライブラリを設計・運用できる。

  • 情報設計

    要件や行動フローをベースに、情報・アクションの整理やオブジェクトモデリングを用いて、サービス全体の画面構造（階層・ナビゲーション・遷移）を体系的に設計し、 UIの骨格となるワイヤーフレームを合理的かつ一貫性のある構造として構築できる。

  • ビジュアルデザイン

    サービスの性質やコンセプトに基づくトンマナ策定や、HIG/Material Design に準拠した設計を行い、視線誘導・可読性・操作性に配慮したUI設計ができる。

  • プロトタイピング

    インタラクションや遷移を備えたプロトタイプを構築し、操作性・設計意図を的確に共有するための検証用アセットとして運用できる。

UX Design

  • ユーザー理解

    ユーザーの行動特性・目的・課題を正確に把握するためのインタビューや簡易リサーチを実施し、 得られた知見を設計の根拠として整理・活用できる。

  • 競合・市場リサーチ

    類似サービスのフロー・情報構造・UIパターンを分析し、共通原則や差別化要素を抽出。 ユーザビリティや業界特性の観点から、設計判断に必要な示唆を得ることができる。

  • 企画・コンセプト定義

    リサーチ結果を基に、サービスの価値提供・体験ゴール・主要タスクを言語化し、 UI/UX設計の方向性となるコンセプトを一貫性のある形で定義できる。

  • 要件定義

    ユーザーの課題やニーズを調査から明確化し、プロダクトの目的やビジネス上の目標と照合しながら、必要な機能要件・非機能要件へ整理できる。

Web Design

Profile

自己紹介

Yuki Nishimura

UIUX Designer / Web Designer

経歴と現在

慶應義塾大学 経済学部卒
受託開発の会社にて、主にコーポレートサイトやサービス紹介のLPなど、Webデザインを中心とした案件に携わりつつ、 アプリケーションUIのデザインにも一部関わってきました。Web領域では構成設計からデザイン、静的コーディングまで一貫して担当し、 フリーランスとしてもコーポレートサイトやLP制作を中心に、要件整理から実装まで幅広く経験してきました。
制作を続ける中で、表層的なビジュアルだけではなく、ユーザーの目的や課題から構造全体を設計する領域により魅力を感じるようになり、 現在は UI/UX デザインを専門として深めていくことを志向しています。

デザインで大事にしていること

デザインを行う際には、意匠的な魅力だけでなく、「ユーザーが迷わず操作できるか」「課題や目的に対して本当に必要な構造か」という視点を常に重視しています。 課題や目的から必要な情報や機能を抽出し、それらを導線・遷移構造・情報の優先度として整理することで、負荷やストレスの少ない体験へと落とし込む設計を心がけています。
また、プロダクトはデザイナーだけでは成立しないため、実装コストや技術的制約への理解を前提に、 エンジニアとのコミュニケーションを密に取りながら現実的で再現性のある UI に仕上げることを大切にしています。

今後の展望

表層的な見た目の改善にとどまらず、ユーザーの目的や課題から本質的な構造を設計し、プロダクトそのものの価値向上に寄与できる UI/UX デザインに取り組んでいきたいと考えています。
また、ユーザー体験の向上だけでなく、事業全体を俯瞰しながらコストとリターンを踏まえた判断を行い、デザインを軸に事業へ貢献できるデザイナーを目指しています。
